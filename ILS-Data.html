Here is an example of the resulting SQL query. Each column represents a layer of the visualization. IDs connect to key words and key words connect to IDs. A special feature of this algorithm I developed is that there are no repeated terms, meaning that an ID that appears in the first column will not appear in the third column. This prevents the incident of a repetitive loop of the same ID and key word in multiple columns. "NULL" is used as a placeholder for an empty entry.





To quantitatively show the efficiency of the new algorithm, I compared the time required to complete the SQL query for the between the old and new algorithm. Within the code, I included a time variable to determine the time elapsed to display the visualization web page. After completing 15 trials for each algorithm I obtained these results. The old algorithm had an average run time of 10.05 seconds while the algoritm had an averge runtime of 1.04 seconds. This clearly shows that the new algorithm is more efficient than the old design.





Here is an example of a visualization for the search term "crystal structure" (note: a smaller data table is being used instead of the COVID-19 data table in this example for testing purposes). Each sphere represents a node (key word or unique ID). Above each node is the key word/ID associated with each node. I added a feature to remove labels with the simple click of a button.
